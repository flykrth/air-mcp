# Security Policy

## Supported Versions

We actively monitor and patch security vulnerabilities in the following versions:

| Version | Supported |
| ------- | --------- |
| 1.0.x   | Yes       |
| < 1.0.0 | No        |

---

## Reporting a Vulnerability

We take the security of AIR-MCP seriously. If you discover a vulnerability, please report it responsibly.

### How to Report
* Do **NOT** open a public issue on GitHub.
* Email your report directly to the security team at `security@air-mcp.org` (or contact the maintainers directly).
* Include as much detail as possible, such as:
  * A description of the vulnerability.
  * The potential impact on the platform.
  * Step-by-step instructions or a proof-of-concept (PoC) script to reproduce the issue.
  * Details of your environment (e.g., local Docker Compose, cloud Supabase keys config, etc.).

### Our Response Process
1. **Acknowledgement**: We will acknowledge receipt of your report within 24 hours.
2. **Triaging**: We will investigate and validate the report within 3 days, keeping you updated on our progress.
3. **Remediation**: If validated, we will draft a patch and verify it against our test suite.
4. **Disclosure**: Once patched, we will release a security advisory and credit you for the discovery.
